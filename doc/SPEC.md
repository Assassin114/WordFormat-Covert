# DocFormatter - Word文档格式批量处理工具

## 1. 项目概述

**项目名称:** DocFormatter
**类型:** 桌面GUI应用程序（Python + PyQt6）
**核心功能:** 对格式混乱的Word文档按模板进行标准化格式化，支持批量处理；提供Word→Markdown转换功能
**目标用户:** 文档编写人员（科技论文、技术报告、军标文档）

---

## 2. 功能模块

### 2.1 模板配置模块

**功能:** 通过GUI配置格式化模板，保存/加载模板文件（JSON格式）

**可配置项:**

| 配置项 | 说明 |
|--------|------|
| 封面信息 | 标题、单位、日期等字段（固定位置） |
| 签署页 | 签名栏、日期栏等 |
| 修改记录 | 章节标题"修改记录"格式 |
| 多级列表 | 1~5级标题的字体、字号、加粗、颜色、行距、段前段后距、缩进 |
| 正文格式 | 字体、字号、行距（1.0/1.5/2.0倍）、首行缩进、对齐方式 |
| 题注格式 | 图序/表序/公式序的编号规则（章节编号或全文统一）、标签格式 |
| 页眉页脚 | 首页不同、奇偶页不同、页码格式 |
| 打印模式 | 单面/双面（影响空白页添加逻辑） |

**特殊排版规则:**
- 图序：位于图下方，必须与上图同页（孤行控制）
- 表序：位于表上方，必须与下表同页
- 公式：居中编号，编号在右侧

### 2.2 文档格式化模块

**功能:** 读取原始Word文档，按模板配置重新应用格式

**处理流程:**
1. 解析原始文档结构和样式
2. 识别并分类内容（封面/正文各级标题/表格/图片/公式/题注）
3. 应用模板配置的格式到对应内容
4. 统一编号：图序/表序/公式序（按章节或全文）
5. 生成/更新目录
6. 处理单双面打印空白页
7. 输出格式化文档

**样式映射策略:**
- 原始文档中Heading 1~5 → 模板1~5级标题格式
- Normal → 正文格式
- 直接格式（font directly set）→ 忽略，统一使用模板定义

### 2.3 批量处理模块

**功能:** 批量选择多个文档，使用同一模板进行格式化

**支持:**
- 文件夹批量选择
- 多文件选择
- 进度显示
- 错误日志记录

### 2.4 Word转Markdown模块

**功能:** 将格式混乱的Word文档转换为Markdown

**转换规则:**
- 标题 → `#` ~ `######`
- 粗体/斜体 → `**` / `*`
- 表格 → Markdown表格
- 图片 → 提取保存，引用路径
- 公式 → LaTeX格式（可选）
- 交叉引用 → 保留为文本占位符

## 3. 技术架构

```
DocFormatter/
├── main.py                    # 程序入口
├── gui/                       # PyQt6 GUI模块
│   ├── main_window.py         # 主窗口
│   ├── template_config.py     # 模板配置界面
│   ├── batch_process.py       # 批量处理界面
│   └── word2md.py             # Word转MD界面
├── core/                      # 核心处理模块
│   ├── formatter.py           # 文档格式化引擎
│   ├── style_mapper.py        # 样式映射
│   ├── numbering.py           # 编号管理（图序/表序/公式序）
│   ├── toc_generator.py       # 目录生成
│   └── word2md_converter.py  # Word转MD转换器
├── templates/                 # 内置模板
├── utils/                     # 工具函数
└── requirements.txt
```

**技术栈:**
- Python 3.10+
- PyQt6（GUI）
- python-docx（Word读写）
- lxml（XML级别样式操作，用于精确格式控制）
- mammoth（Word→MD转换，可复用已安装的markitdown）

---

## 4. 输入输出

**输入:**
- 原始Word文档（.docx）
- 模板配置文件（.json）

**输出:**
- 格式化后的Word文档（.docx）
- 处理报告（.log）
- Markdown文件（.md，仅转换模式）

## 6. 验收标准

1. ✅ 模板配置可保存/加载，JSON格式
2. ✅ 5级标题格式正确应用（字体/字号/行距/段前段后）
3. ✅ 图序在图下方且与图同页
4. ✅ 表序在表上方且与表同页
5. ✅ 图/表/公式编号正确（章节编号：X-Y格式）
6. ✅ 目录按封面→签署页→修改记录→正文顺序生成
7. ✅ 单面/双面打印模式正确添加空白页
8. ✅ 批量处理多个文档
9. ✅ Word→Markdown转换保留结构

---

## 7. 风险与依赖

**风险:**
- 600页文档内存占用较高 → 考虑流式处理
- 直接格式与样式混杂 → 以样式为准，直接格式忽略
- python-docx对编号列表（multilevel）控制有限 → 需直接操作OOXML

**依赖:**
- PyQt6
- python-docx
- lxml
- mammoth（Word→MD）
