# 页眉页脚规格说明

## 1. 页面分类

文档按内容分为三类页面，每类有不同的页眉页脚设置：

| 页面类型 | 内容 | 页眉 | 页脚 | 页码 |
|---------|------|------|------|------|
| 第1类 | 封面页、签署页、修改记录页 | 统一内容 | 无 | 无 |
| 第2类 | 目录页 | 统一内容 | 有 | 大写罗马数字（I, II, III...）从1开始 |
| 第3类 | 正文部分 | 统一内容 | 有 | 大写罗马数字从1开始（新的一组） |

## 2. 技术实现

### 2.1 分节控制
Word文档通过**分节符**分隔不同页面类型，每一节可以有独立的页眉页脚设置。

### 2.2 节类型识别
根据节的位置和内容判断类型：
- **第1节**（文档开头）：封面/签署页/修改记录页
- **包含"目录"标题的节**：目录页
- **其他节**：正文部分

### 2.3 页码格式
```python
# 罗马数字格式
I, II, III, IV, V, VI, VII, VIII, IX, X, ...
```

### 2.4 页眉内容
- 第1类：可以是文档标题或"机密★"等标识
- 第2类：通常同第1类
- 第3类：通常同第1类

## 3. OOXML实现

### 3.1 节属性设置
```xml
<!-- 设置节类型 -->
<w:sectPr>
    <w:headerReference w:type="first" r:id="rId1"/>
    <w:footerReference w:type="first" r:id="rId2"/>
    <w:type w:val="nextPage"/>  <!-- 分节符类型 -->
</w:sectPr>
```

### 3.2 页码域
```xml
<w:fldSimple w:instr=" PAGE \* roman ">
    <w:r><w:t>1</w:t></w:r>
</w:fldSimple>
```

## 4. 实现步骤

1. 解析文档结构，识别各节类型
2. 为每类节创建独立的页眉页脚
3. 设置页码格式和起始值
4. 确保节之间的分节符正确

## 5. 验收标准

- [ ] 第1类页面：无页码
- [ ] 第2类页面：大写罗马数字页码，从I开始
- [ ] 第3类页面：大写罗马数字页码，从I开始（新编号）
- [ ] 各类型页眉内容正确
